# Session 2: Notion MCP preference in conversation mode

## Goal

Update the system prompt for conversation mode so Promaia prefers the Notion MCP over internal query tools when fetching single pages.

## Rationale

- The Notion MCP always has the latest data (it hits the remote API directly)
- Internal query tools (vector search, SQL) are snapshots — great for bulk/pattern queries but potentially stale for individual pages
- Promaia should treat the remote as the authoritative "head"
- Internal tools are highly efficient for pulling larger quantities of data for expansive context and pattern recognition

## Changes needed

- Add system prompt instructions in conversation mode that:
  - For single-page lookups: prefer Notion MCP (most accurate, always fresh)
  - For multi-page queries, pattern matching, bulk context: use internal query tools (faster, more efficient)
  - Frame it as: remote = accuracy/freshness, local = breadth/speed
